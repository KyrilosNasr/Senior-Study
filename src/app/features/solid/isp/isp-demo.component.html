SOLID Principles

Interface Segregation Principle

Clients should not be forced to depend upon interfaces that they do not use.

What is ISP?

The Interface Segregation Principle states that no client should be forced to depend on methods it does not use. Instead of having one large interface, we should have multiple smaller, more specific interfaces. This prevents classes from being forced to implement methods they don't need.

Key Benefits

  • Prevents fat interfaces with unused methods
  • Reduces coupling between classes
  • Makes interfaces more focused and cohesive
  • Easier to maintain and extend

Key Takeaways

  • Split large interfaces into smaller, focused ones
  • Classes should only implement interfaces they actually use
  • Avoid "fat" interfaces with many unrelated methods
  • Use composition to combine multiple interfaces when needed
