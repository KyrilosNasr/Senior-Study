Liskov Substitution Principle (LSP)

Objects of a superclass should be replaceable with objects of its subclasses without breaking the application.

What is LSP?

The Liskov Substitution Principle states that derived classes must be substitutable for their base classes. This means that any code that works with a base class should also work with any of its derived classes without modification. The derived class should not change the expected behavior of the base class.

Key Benefits:

  • Ensures proper inheritance relationships
  • Prevents unexpected behavior in polymorphic code
  • Makes code more predictable and reliable
  • Enables safe use of polymorphism

Key Takeaways

  • ✓ Subtypes must honor the contract of their base types
  • ✓ Don't override methods in a way that changes expected behavior
  • ✓ Use composition over inheritance when LSP cannot be maintained
  • ✓ Preconditions cannot be strengthened, postconditions cannot be weakened
