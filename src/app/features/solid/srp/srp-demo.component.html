SOLID Principles

Single Responsibility Principle

A class should have only one reason to change, meaning it should have only one job or responsibility.

What is SRP?

The Single Responsibility Principle states that a class should have only one reason to change. This means each class should focus on doing one thing well. When a class has multiple responsibilities, it becomes harder to maintain, test, and understand.

Key Benefits

  • Easier to understand and maintain
  • Reduced coupling between classes
  • Better testability
  • Improved code reusability
