Subject Patterns

Using BehaviorSubject, ReplaySubject, and AsyncSubject for state management and caching.

Subject Types

Subjects are special observables that can both emit values and be subscribed to. Different types serve different purposes.

Subject Types:

  • BehaviorSubject: Emits current value to new subscribers
  • ReplaySubject: Replays last N values to new subscribers
  • AsyncSubject: Emits only last value when completed

Key Takeaways

  • ✓ Use BehaviorSubject for current state that new subscribers need
  • ✓ Use ReplaySubject to cache and replay recent values
  • ✓ Use AsyncSubject for one-time async operations
  • ✓ Expose observables, not subjects, to prevent external mutation
