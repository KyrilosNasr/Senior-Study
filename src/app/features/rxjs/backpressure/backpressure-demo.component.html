Backpressure & Rate Limiting

Controlling the frequency of emissions to handle high-frequency events efficiently.

What is Backpressure?

Backpressure occurs when a producer emits values faster than a consumer can process them. Rate limiting operators help control this flow.

Rate Limiting Operators

  • debounceTime: Wait for pause, then emit latest
  • throttleTime: Emit first, then ignore for duration
  • sampleTime: Emit latest value at intervals
  • auditTime: Wait for source to complete, emit latest

Key Takeaways

  • Use debounceTime for search inputs and form validation
  • Use throttleTime for scroll events and mouse movements
  • Use sampleTime when you need periodic updates
  • Choose appropriate delay times based on use case
