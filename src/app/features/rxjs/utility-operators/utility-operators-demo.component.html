Utility Operators

Utility operators provide side effects, cleanup, and timing control without modifying the observable stream. They're essential for logging, loading states, and resource cleanup.

Key Operators:

  • tap(): Side effects without modifying stream - Logging, debugging
  • finalize(): Cleanup logic that always executes - Hide loaders, cleanup
  • delay(): Delay emissions by specified time - Testing, animations
  • timeout(): Error if doesn't complete in time - Prevent hanging requests

Key Takeaways

  • ✓ Use tap() for side effects (logging, debugging) without modifying the stream
  • ✓ Always use finalize() for cleanup (hide loaders, close connections) - it always executes
  • ✓ Use timeout() to prevent hanging requests - improves UX
  • ✓ Use delay() for testing, animations, or rate limiting
  • ✗ Never put side effects in subscribe() - use tap() instead for better composition
