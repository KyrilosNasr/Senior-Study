Code Examples

Basic Signal

// Create a signal
count = signal(0);

// Read signal value
const value = count();

// Update signal
count.set(10);
count.update(value => value + 1);

Computed Signal

// Computed from other signals
doubleCount = computed(() => this.count() * 2);

// Complex computed
summary = computed(() => ({
  count: this.count(),
  double: this.doubleCount(),
  name: this.name()
}));

Effects

// Effect with side effects
effect(() => {
  const count = this.count();
  if (count > 10) {
    console.log('Count exceeded 10!', count);
  }
});

Signals with Dynamic Components 

// Signal-based configuration
formConfig = signal<DynamicFormConfig>({
  fields: [...],
  submitLabel: 'Submit'
});

// Use in template
<app-dynamic-form [config]="formConfig()" />

Key Takeaways

  • Signals provide fine-grained reactivity for Angular applications
  • Use computed() for derived values that depend on other signals
  • Effects run automatically when their dependencies change
  • Signals integrate seamlessly with Angular's change detection
